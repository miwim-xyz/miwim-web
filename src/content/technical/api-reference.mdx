# API Reference

> <span className="coming-soon-badge">Coming Q4 2026</span> The Miwim white-label API is in the design phase. This page outlines the **planned** API surface.

## Planned Endpoints

### Subscription Management
```
POST /api/v1/subscribe          Create a new subscription
GET  /api/v1/subscribe/{id}     Get subscription details
PUT  /api/v1/subscribe/{id}     Update subscription plan
DEL  /api/v1/subscribe/{id}     Cancel subscription
```

### Node Access
```
GET  /api/v1/nodes              List available nodes
GET  /api/v1/nodes/{id}         Get node details and health
GET  /api/v1/config/{token}     Get Singbox config for subscription
```

### Usage & Billing
```
GET  /api/v1/usage              Get bandwidth usage stats
GET  /api/v1/usage/daily        Daily breakdown
GET  /api/v1/billing            Billing history
POST /api/v1/billing/topup      Add credit (fiat or token)
```

### IP Management (Edge)
```
GET  /api/v1/ips                List available IP pools
POST /api/v1/ips/bind           Bind a dedicated residential IP
DEL  /api/v1/ips/bind/{id}      Release a dedicated IP
GET  /api/v1/ips/rotate         Get a fresh IP from pool
```

## Planned Authentication

All API requests will require a Bearer token:

```
Authorization: Bearer {api_key}
```

API keys will be issued to white-label clients based on their $MIWIM staking tier:

| Tier | Staking Requirement | Rate Limit | Features |
|------|-------------------|------------|----------|
| Basic | 10,000 $MIWIM | 100 req/min | Backbone access |
| Pro | 50,000 $MIWIM | 1,000 req/min | Backbone + Edge |
| Enterprise | 200,000 $MIWIM | 10,000 req/min | Full access + dedicated pools |
