# Provider Agent

> <span className="coming-soon-badge">Planned Architecture</span> The provider agent is under active development. This page describes the **design specification**, not a live system.

The Miwim provider agent will be a lightweight Go binary that runs on provider machines, managing the Singbox proxy service and reporting health data to the platform.

## Planned System Requirements

| Requirement | Minimum | Recommended |
|-------------|---------|-------------|
| OS | Linux (Ubuntu 20.04+, Debian 11+) | Ubuntu 22.04 LTS |
| CPU | 1 vCPU | 2 vCPU |
| RAM | 512 MB | 1 GB |
| Bandwidth | 50 Mbps | 200+ Mbps |
| Storage | 1 GB | 5 GB |
| Network | Outbound port 443 open | All outbound open |

## Planned Configuration

```yaml
# /etc/miwim/config.yaml (planned schema)
platform:
  api_url: https://api.miwim.io
  node_id: "uuid-assigned-at-registration"
  auth_token: "jwt-token-from-registration"

agent:
  singbox_path: /usr/local/bin/sing-box
  singbox_config: /etc/miwim/singbox.json
  metrics_port: 9090
  heartbeat_interval: 30s
  benchmark_interval: 6h

# For residential nodes only:
tunnel:
  enabled: true
  relay_address: relay-hk.miwim.io:443
  protocol: vless-ws
  uuid: "tunnel-uuid"
  keepalive: 15s
```

## Heartbeat Protocol (Design)

The agent will send health data to the platform every 30 seconds:

```
POST /api/nodes/{id}/heartbeat
Authorization: Bearer {auth_token}

{
  "timestamp": "...",
  "status": "active",
  "uptime_seconds": 86400,
  "cpu_usage_pct": 12.5,
  "memory_usage_pct": 34.2,
  "bandwidth_up_mbps": 95.4,
  "bandwidth_down_mbps": 210.8,
  "active_connections": 42,
  "latency_ms": { "CN": 45, "JP": 12, "US": 180, "EU": 95, "SG": 30 },
  "agent_version": "0.x.x",
  "binary_hash": "sha256:..."
}
```

## Benchmark Routine (Design)

Planned to run on startup and every 6 hours:

1. Latency test — ping 10 regional endpoints
2. Bandwidth test — iperf3-style up/down measurement
3. IP classification — query ipinfo.io API
4. Route quality — traceroute to 5 targets
5. Binary attestation — compute and report SHA256
6. Report results to platform

## Reverse Tunnel (Residential Nodes)

When `tunnel.enabled: true`, the agent will:

1. Initiate outbound VLESS/WS connection to assigned relay on port 443
2. Connection looks like HTTPS traffic — ISPs won't block it
3. Tunnel stays persistent with 15s keepalive
4. Auto-reconnect on drop with exponential backoff (1s, 2s, 4s, max 60s)
5. After 5 consecutive failures: alert platform, mark node as degraded

## Agent Lifecycle (Design)

```
Start → Run benchmark → Register with platform → Start Singbox
  → Enter heartbeat loop (every 30s)
  → Re-benchmark (every 6h)
  → If config update received → restart Singbox with new config
  → If SIGTERM → graceful shutdown (drain connections, notify platform)
```
