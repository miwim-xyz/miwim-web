# Run a Node

Contributing bandwidth to the Miwim network earns you $MIWIM token rewards. Here's everything you need to know.

## Requirements

| Requirement | Minimum | Recommended |

## Quick Start

### 1. Apply as a Genesis Node
Visit [miwim.io](/#waitlist) and fill out the provider application form. We'll review your application and send credentials.

### 2. Install the Agent
```bash
curl -fsSL https://install.miwim.io | bash
```

### 3. Configure
Edit `/etc/miwim/config.yaml` with your credentials:
```yaml
platform:
  api_url: https://api.miwim.io
  node_id: "your-node-id"
  auth_token: "your-auth-token"
```

### 4. Start
```bash
sudo systemctl start miwim-agent
sudo systemctl enable miwim-agent  # auto-start on boot
```

### 5. Verify
```bash
sudo systemctl status miwim-agent
curl http://localhost:9090/metrics
```

## FAQ

**Do I need to stake tokens during beta?**
No. Staking requirement begins at mainnet token launch (Q1 2027). Beta providers earn future airdrop allocation instead.

**Can I run multiple nodes?**
Yes. Each node needs its own credentials and must run on a separate IP address.

**What happens if my node goes offline?**
Brief outages (< 10 minutes) are tolerated. Extended downtime affects your health score and may reduce rewards. Users are automatically failed over to other nodes.

**Is my home IP exposed to users?**
No. Residential nodes use reverse tunnels — users connect to a relay, which forwards traffic through the tunnel. Your home IP is never in user configurations.
